# 行业标准与规范速查

## 等级保护（网络安全等级保护）

### GB/T 22239-2019（等保2.0）
- **第一级（自主保护）**：基本安全要求，适用于一般信息系统
- **第二级（指导保护）**：在GB/T 22239中增加了安全审计、通信保密等要求
- **第三级（监督保护）**：适用于重要信息系统，要求较全面。关键控制点：
  - 安全物理环境：机房出入口控制、温湿度控制、电力供应
  - 安全通信网络：网络架构、通信传输加密、可信验证
  - 安全区域边界：边界防护、访问控制、入侵防范、恶意代码防范
  - 安全计算环境：身份鉴别、访问控制、安全审计、入侵防范、数据完整性/保密性
  - 安全管理中心：系统管理、审计管理、安全管理、集中管控
  - 安全管理制度：策略、制度、规程、记录
  - 安全管理机构：岗位设置、人员配备、授权和审批
  - 安全管理人员：录用、培训、考核、离岗
  - 安全建设管理：定级备案、方案设计、产品采购、软件开发、工程实施
  - 安全运维管理：环境管理、资产管理、介质管理、设备维护、漏洞和风险管理
- **第四级（强制保护）**：适用于关键信息基础设施等极高要求系统
- **第五级（专控保护）**：适用于极端重要系统

### 等保实施流程
定级 → 备案 → 建设整改 → 等级测评 → 监督检查

## ISO/IEC 27001:2022

### 控制域（Annex A, 93项控制）
- A.5 组织控制（37项）：信息安全策略、威胁情报、ICT就绪
- A.6 人员控制（8项）：筛选、意识培训、纪律程序、保密协议
- A.7 物理控制（14项）：物理安全边界、设备安全、清桌清屏
- A.8 技术控制（34项）：端点设备、访问控制、加密、安全编码

### 认证流程
范围界定 → 差距分析 → ISMS建立与实施 → 内部审核 → 管理评审 → 第一阶段审核 → 第二阶段审核 → 获证 → 监督审核（每年）→ 再认证（三年）

## NIST 网络安全框架 v2.0

### 六大功能
- **治理（GOVERN）**：组织环境、风险管理策略、角色与职责、策略与监督
- **识别（IDENTIFY）**：资产管理、风险评估、改进
- **保护（PROTECT）**：身份管理、意识培训、数据安全、平台安全、技术基础设施韧性
- **检测（DETECT）**：持续监控、不利事件分析
- **响应（RESPOND）**：事件管理、事件分析、事件响应报告与沟通、事件缓解
- **恢复（RECOVER）**：事件恢复计划执行、事件恢复沟通

## PCI DSS v4.0

### 12项核心要求
1. 安装和维护网络安全控制
2. 对系统配置应用安全配置标准
3. 保护存储的账户数据
4. 通过加密网络传输保护持卡人数据
5. 保护系统和软件免受恶意软件侵害
6. 开发并维护安全的系统和软件
7. 按业务需要限制对系统组件和持卡人数据的访问
8. 识别用户并验证对系统组件的访问
9. 限制对持卡人数据的物理访问
10. 记录和监控对系统组件和持卡人数据的所有访问
11. 定期测试系统和网络的安全性
12. 通过组织策略和计划支持信息安全

### SAQ类型
- SAQ A：外包卡数据处理的电商
- SAQ A-EP：部分外包的电商
- SAQ B：仅使用 imprint 机器的商户
- SAQ B-IP：使用独立终端的商户
- SAQ C：使用互联网连接的支付终端
- SAQ C-VT：虚拟终端
- SAQ D：所有其他商户 + 所有服务提供商

## 行业特定标准

### 金融行业
- **JR/T 0071**：金融行业信息安全等级保护实施指引
- **JR/T 0197**：金融数据安全 数据安全分级指南
- **银保监发[2020]27号**：银行保险机构信息科技外包风险监管办法

### 政务/信创
- **GB/T 39786**：信息安全技术 信息系统密码应用基本要求（密评）
- **信创产品目录**：CPU/OS/数据库/中间件国产化要求

### 运营商
- **YD/T 系列标准**：电信和互联网行业网络安全相关标准

### 医疗
- **国家健康医疗大数据标准、安全和服务管理办法**
- **医院信息系统等级保护基本要求**

### 工业控制
- **GB/T 36323**：信息安全技术 工业控制系统信息安全分级规范
- **GB/T 37962**：信息安全技术 工业控制系统产品信息安全
