# 商务实践速查

## 咨询项目计费模式

### 1. 人天计费
- **常规咨询顾问**：1,600 - 3,000 元/人天
- **高级咨询顾问**：3,000 - 5,000 元/人天
- **资深专家/合伙人**：5,000 - 8,000 元/人天
- **驻场服务**：通常上浮 20-30%

### 2. 固定总价
适用场景：范围明确、交付物清晰的项目
- 渗透测试（单个Web系统）：2-8万
- 渗透测试（包含APP+Web+内网）：8-20万
- 等保咨询（三级，单个系统）：8-30万
- 安全规划/架构设计：30-100万+
- 数据安全治理规划：50-200万+
- 应急响应（单次）：3-15万
- 代码审计（10万行代码级别）：5-15万
- 红蓝对抗演练（一周）：15-50万

### 3. 年框/订阅
- 安全顾问年框：20-100万/年
- MSS（托管安全服务）：根据资产规模和SLA级别定价
- 安全运营外包：按监控资产数量和响应级别计价

### 4. 成果付费
- 通常用于众测/漏洞悬赏
- 按漏洞等级计费：严重漏洞 5,000-50,000元/个
- 合同需明确漏洞等级定义和验证标准

## 人天估算

### 典型项目人天参考

| 项目类型 | 规模 | 人天范围 |
|---------|------|---------|
| 渗透测试（Web系统） | 1-3个系统 | 10-25人天 |
| 渗透测试（含APP+内网） | 综合 | 25-50人天 |
| 等保咨询（三级） | 1个系统 | 30-60人天 |
| 安全规划方案 | 企业级 | 20-50人天 |
| 代码审计 | 10万行 | 15-25人天 |
| 安全培训（定制） | 3天课程 | 10-20人天 |
| 红蓝对抗 | 1周 | 30-60人天 |
| 数据安全评估 | 企业级 | 20-40人天 |

### 人天估算考虑因素
- 项目复杂度（技术栈多样、系统间耦合度）
- 客户配合度（是否有专人对接、信息获取难易）
- 地域因素（是否需要差旅）
- 安全级别要求（是否涉密、是否需现场作业）
- 文档要求（报告详细程度、是否需要多版本）

### 项目团队配置比例
- 项目经理：15-20%工作量
- 高级顾问/架构师：30-40%工作量
- 执行顾问：40-50%工作量
- 技术专家（按需）：10-20%工作量

## 招投标流程

### 政府采购常见方式
1. **公开招标**：公告邀请不特定供应商，竞争最充分
2. **邀请招标**：向3家以上特定供应商发邀请
3. **竞争性谈判**：与3家以上供应商分别谈判
4. **竞争性磋商**：类似竞争性谈判但更注重方案质量
5. **单一来源**：只能从唯一供应商采购（须论证）
6. **询价**：向3家以上供应商询价，最低价中标

### 投标文件标准结构
```
第一部分：商务标
  - 投标函
  - 法定代表人授权书
  - 营业执照、资质证书
  - 财务报表/纳税证明
  - 同类项目业绩（合同复印件）
  - 人员资质（CISP/CISSP/CISA等）

第二部分：技术标
  - 项目理解与需求分析
  - 技术方案（方法论、实施路径）
  - 项目实施方案（计划、人员配置）
  - 质量保证措施
  - 售后服务承诺

第三部分：价格标
  - 报价一览表
  - 分项报价明细
  - 价格构成说明
```

### 评分常见结构
- 价格分：30-40%
- 技术分：40-50%
- 商务分：10-20%

### 关键注意事项
- 实质性响应：必须逐条响应招标文件要求，缺项直接废标
- 资质门槛：CISP/CISSP/CISAW等安全资质是常见门槛条件
- 业绩要求：通常要求近3年同类项目业绩
- 联合体投标：注意联合体各方的资质是否满足要求
- 保证金：投标保证金、履约保证金（通常为合同额的5-10%）

## 合同关键条款关注点

### 服务范围（SOW）
- 清晰定义包含和不包含的服务内容
- 避免模糊表述（"等""相关系统"需要明确定义）

### 交付与验收
- 交付物清单及格式要求
- 验收标准和验收流程
- 验收不通过的整改机制和时限

### 知识产权
- 交付物（报告、方案、代码）的知识产权归属
- 使用的方法论和工具的知识产权
- 保密信息的使用限制

### 免责条款
- 渗透测试引起的系统异常（需提前约定）
- 第三方组件漏洞的修复责任界定
- 不可抗力事件的免责

### 付款条款
- 常见节奏：3-3-3-1（签约30% + 中期30% + 验收30% + 质保金10%）
- 或：5-4-1（签约50% + 验收40% + 质保金10%）
- 质保期：通常1年
