# CSCA：网络安全与AI咨询专家 Skill

CSCA（Cyber Security Consulting Agent）是一个 Claude Code Skill，把 Claude 变成网络安全咨询交付专家：覆盖安全评估、渗透测试、等保合规、安全建设方案、培训课件、投标报价等网安咨询交付场景，同时覆盖 AI 双向咨询（AI 赋能安全 + 安全赋能 AI）。

交付物执行「分析 → 规划 → 生成 → 审查」四阶段流程，输出前强制通过一次写前检查和三道对抗关卡，遵循去AI化写作规范，目标是可以直接交付客户的完成品。

## 能力覆盖

**网安咨询**
安全评估 / 渗透测试 / 代码审计 / 日志分析 / 数据安全 / 等保与合规审计 / 风险评估 / 应急响应 / 安全建设方案 / 安全架构 / 安全培训与 CTF 课件 / 报价投标与预算编制 / 安全制度与咨询报告

**AI 赋能安全（AI for Security）**
AI 威胁检测 / SOC Copilot / AI 安全运营 / AI 代码审计 / AI 渗透测试 / AI 威胁情报 / AI 钓鱼检测 / AI 与 SOAR、SIEM 集成

**安全赋能 AI（Security for AI）**
OWASP LLM Top 10 评估 / 大模型红队测试 / AI 合规与算法备案 / AI 战略治理与选型 / RAG 与 Agent 架构设计

## 质量机制（v3.1）

- **四阶段强制**：每个交付物必经分析、规划、生成、审查，每阶段有门禁块，不可跳过
- **三道对抗关卡**：写后自审（逐段排查记录）→ 事实核查（10-15 个关键断言逐条验证）→ 评分挑战（9 维度评分）
- **评分规则**：全部维度 ≥ 9 且算术平均 > 9.1 方可交付；首轮平均 ≤ 8.0 强制重写；第 2 轮起提分必须有本轮实际修改证据，无修改不提分
- **可实施性硬门槛**：分阶段实施路径 + 资源需求估算 + 验收标准，缺一直接不通过
- **去AI化写作规范**：160+ 禁用词（含口语词与 AI 符号）、表格去AI化、句式多样性、L1/L2/L3 语体分级、视觉格式规范
- **审查日志**：全部审查过程写入 `csca-review-log-YYYYMMDD.md`，对话内保持精简输出，过程可回溯
- **轻量级任务豁免**：纯信息查询、小段润色、单点咨询走简化路径，不触发完整流程

## 安装

将 `csca/csca` 文件夹（或直接解压 `csca.zip`）复制到 Claude Code 的 skills 目录：

**个人级（所有项目可用）**，Windows PowerShell：

```powershell
Copy-Item -Recurse ".\csca\csca" "$env:USERPROFILE\.claude\skills\csca"
```

**项目级（仅当前项目可用）**：

```powershell
Copy-Item -Recurse ".\csca\csca" ".\.claude\skills\csca"
```

安装后重启 Claude Code 会话生效。

## 使用

**手动触发**：输入 `/csca`，或说「启动CSCA」「用咨询专家模式」。

**自动触发**：描述涉及网安咨询或 AI 安全咨询的交付需求即可，例如：

- 「帮我写一份金融行业的渗透测试报告」
- 「做一个等保三级整改建设方案，政务云客户」
- 「评估我们的大模型应用，按 OWASP LLM Top 10」
- 「出一份安全意识培训方案和预算」

**审查日志**：实质性交付物的审查过程记录会写入会话当前工作目录下的 `csca-review-log-YYYYMMDD.md`，包含逐段排查记录、断言验证表、每轮评分与提分证据。交付物末尾附日志路径。

## 合规边界

- 不输出可直接执行的攻击脚本、exploit 代码、恶意软件
- 渗透测试方案以授权前提和合法合规为基础
- 客户专有数据无法获得时以 `[待客户提供]` 标记，不编造数字
- 不确定的内容明确说明，不模糊处理
